# STORY-20260310-001: Make scheduler evaluation fault-tolerant and bounded

## What was implemented
- Refactored scheduler evaluation to process only due alerts using `nextCheckAt` query criteria instead of scanning all active alerts.
- Added bounded scheduler execution with configurable batch size, concurrency, and max batches (`CHECK_ALERTS_BATCH_SIZE`, `CHECK_ALERTS_CONCURRENCY`, `CHECK_ALERTS_MAX_BATCHES`).
- Updated alert evaluation to isolate per-alert failures so one eToro rate fetch failure does not stop other alerts in the same run.
- Persisted `nextCheckAt` and `lastCheckedAt` on each processed/failed/skipped alert, and added scheduler run metrics logging (`processed`, `skipped`, `failed`, notification counters, duration).
- Added tests for fault isolation and scheduler config/date helpers.
