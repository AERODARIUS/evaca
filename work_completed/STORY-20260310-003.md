# STORY-20260310-003: Add pagination and hard limits to alert/notification reads

## What was implemented
- Added backend cursor-based pagination to `listUserNotifications` using Firestore `orderBy + limit + startAfter` with hard page-size limits.
- Added pagination token helpers (`encodePageToken`, `decodePageToken`) and request page-size normalization on backend.
- Added frontend cursor-based alert reads via `listAlertsPage` with max page size and deterministic cursoring.
- Updated app alert loading flow to incremental pagination with `Load more alerts` UX and explicit loading states.
- Updated alert writes (`create/update/toggle`) to persist `nextCheckAt` so scheduler and reads align with bounded-query contracts.
- Added tests for backend page token helpers and frontend first/next/empty page behavior.
