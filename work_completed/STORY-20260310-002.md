# STORY-20260310-002: Replace in-memory API throttling with distributed limits

## What was implemented
- Replaced process-local in-memory rate limiting with Firestore transaction-based distributed counters in `_rateLimits`.
- Added deterministic document-keying by scope/endpoint/identity/window (`toRateLimitDocId`) to enforce limits consistently across instances.
- Enforced distributed throttling for endpoint, uid, and IP dimensions via `enforceRequesterRateLimits`.
- Updated callable and HTTP auth flows to await distributed limit checks before serving market-data and notification listing requests.
- Added unit tests for distributed key generation and distributed-limit pass/fail behavior.
