# STORY-20260312-002: Add resilient UI error handling for alert workflows

## What was implemented
- Added load-failure handling in `App` for alert listing and pagination with actionable retry.
- Added deterministic mutation-state handling in `AlertsTable` for delete/toggle operations:
  - in-flight guard to prevent duplicate actions
  - surfaced error banner with retry action
  - guarded/disabled controls during mutation
- Added stale-data warning in table when latest load failed.
- Added/extended integration tests to cover transient load failure retry and mutation failure surfacing.
