# STORY-20260309-001: Lock down market-data function access

## What was implemented
- Enforced authenticated caller + App Check requirements for callable market-data functions.
- Added `enforceAppCheck: true` to callable handlers and explicit guard checks for `request.auth.uid` and `request.app.appId`.
- Protected HTTP market-data wrappers by requiring both Firebase ID token (`Authorization: Bearer`) and App Check token (`x-firebase-appcheck`) verification.
- Added per-caller throttling guardrails using in-memory rate limiting (per uid and per IP for HTTP).
- Added structured rejection logs with caller uid (or `anonymous`) and rejection reason.
